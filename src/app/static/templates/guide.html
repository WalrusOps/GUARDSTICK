GuardStick Security Monitoring System

GuardStick is a comprehensive security monitoring and analysis system that combines real-time system monitoring, automated security tasks, and AI-powered analysis to protect your system.

Installation

Prerequisites

  • Python 3.8 or higher
  • 16GB RAM minimum (for LLM operations)
  • 50GB free disk space
  • Unix-based operating system (macOS or Linux)

Quick Start

# Clone the repository
git clone https://github.com/yourusername/guardstick.git
cd guardstick

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Unix/macOS

# Install dependencies
pip install -r requirements.txt

# Start the application
python src/app.py

Key Features

Real-time Monitoring

GuardStick provides continuous monitoring of:

  • CPU usage and performance metrics
  • Memory utilization and management
  • Disk usage and storage analysis
  • Network connections and traffic patterns

Security Analysis

  • Automated security scans
  • Malware detection
  • System integrity verification
  • Firewall management
  • Privacy protection features

AI-Powered Insights

Leveraging the Mistral-7B language model for:

  • Log analysis and interpretation
  • Threat detection and assessment
  • Security recommendations
  • Pattern recognition in system behavior

Getting Started

Basic Configuration

Create a .env file in your project root:

FLASK_APP=src/app.py
FLASK_ENV=development
MODEL_PATH=/models/Mistral-7B-v0.3
PORT=5002

First Run

After installation, access the web interface at:

http://localhost:5002

Frequently Asked Questions

General Questions

Q: How often should I run security scans?
A: We recommend running basic security scans daily and comprehensive scans weekly.

Q: Can I customize security checks?
A: Yes, you can modify existing scripts or add new ones in the scripts directory.

Q: How are logs stored?
A: Logs are stored in the data/log_reports directory with encryption for sensitive data.

Technical Questions

Q: What permissions are required?
A: GuardStick needs administrator privileges for system-level operations.

Q: Can I use a different LLM model?
A: Yes, you can modify the LLM configuration in llm_api.py to use different models.
