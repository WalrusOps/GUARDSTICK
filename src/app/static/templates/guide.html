Introduction

GuardStick is a powerful security tool designed to safeguard your system through continuous monitoring and analysis. It offers an intuitive web interface for easy navigation and provides detailed insights into your system's security posture.

Installation

Prerequisites

Before installing GuardStick, ensure your system meets the following requirements:

  • Python 3.8 or higher
  • 16GB RAM minimum (for AI model operations)
  • 50GB free disk space
  • Unix-based operating system (macOS or Linux)

Quick Start Guide

Follow these steps to install and run GuardStick:

# Clone the repository
git clone https://github.com/yourusername/guardstick.git
cd guardstick

# Create a virtual environment
python -m venv venv
source venv/bin/activate  # On Unix/macOS

# Install dependencies
pip install -r requirements.txt

# Start the application with admin privileges
sudo python src/app/app.py

Key Features

  • Real-time Monitoring: Keep track of CPU, memory, disk usage, and network connections.
  • Security Analysis: Perform automated security scans, malware detection, and system integrity checks.
  • AI-Powered Insights: Utilize the Mistral-7B language model for log analysis, threat detection, and security recommendations.
  • User-Friendly Interface: Access all features through an intuitive web interface.

Getting Started

Basic Configuration

Create a .env file in the project root directory with the following content:

FLASK_APP=src/app.py
FLASK_ENV=development
MODEL_PATH=/models/Mistral-7B-v0.3
PORT=5002

This configuration sets up the Flask application, environment, AI model path, and the port number for the web interface.

First Run

After installation, start the application:

sudo python src/app/app.py

Then, open your web browser and navigate to:

http://localhost:5002

You should see the GuardStick Dashboard.

System Components

Dashboard

The Dashboard provides an overview of your system's status, including:

  • System Status Overview: Quick glance at CPU, memory, disk, and network usage.
  • Real-Time Metrics: Live graphs showing performance data over time.
  • Recent Security Events: List of the latest security scans and findings.
  • Quick Actions: Buttons to run immediate scans or access critical features.

Security Tasks

Under Security Tasks, you can perform various security checks:

  • Malware Scanning: Detect and remove malicious software.
  • Firewall Verification: Ensure your firewall is active and properly configured.
  • Privacy Checks: Review application permissions and privacy settings.
  • System Configuration Analysis: Analyze system settings for potential vulnerabilities.

Log Analysis

The Log Analysis section allows you to:

  • View detailed logs generated by security scans.
  • Search and filter logs based on keywords or date ranges.
  • Download or delete logs as needed.

LLM Analysis

LLM Analysis leverages AI to provide insights:

  • Interpret complex log data.
  • Identify patterns and anomalies.
  • Receive security recommendations based on log analysis.

Using GuardStick

Running Security Scans

  1. Navigate to Security Tasks from the sidebar.
  2. Select the desired scan by clicking on its card.
  3. Click the Run Scan button.
  4. Monitor the scan status; it updates to Complete when finished.
  5. View the results directly below the scan or in the Log Analysis section.

Analyzing Logs

  • Access Log Analysis from the sidebar.
  • Use the search bar to find specific logs.
  • Click on a log file to view its contents.
  • Apply filters or search within the log for specific entries.

AI-Powered Insights

  1. Go to LLM Analysis to use AI for interpreting logs.
  2. Select logs for analysis using the checkboxes.
  3. Type in your question or request in plain language.
  4. Click Analyze and wait for the AI-generated insights.

Detailed Scan Guide

Core Security Scans

1. Malware Check (1-check-malware.py)

What it does:

  • Scans applications for known malware signatures.
  • Verifies application integrity using cryptographic hashes.
  • Identifies tampered or malicious software.

Why it's important:

  • Protects against harmful software that can compromise your system.
  • Ensures that all applications are legitimate and safe to use.
  • Detects potential security threats before they cause damage.

When to use:

  • Daily, especially after installing new software.
  • When experiencing unusual system behavior.
  • After downloading files from untrusted sources.

2. Firewall Analysis (2-check-firewall.py)

What it does:

  • Checks the status of your firewall.
  • Reviews application network permissions.
  • Identifies open ports and potential vulnerabilities.

Why it's important:

  • Blocks unauthorized access to your system.
  • Manages which applications can send and receive data.
  • Strengthens your network security posture.

When to use:

  • Weekly.
  • After installing network-related applications.
  • When experiencing network connectivity issues.

Data Collection Scans

101. System Information Collection (101-collect-systeminfo.py)

What it does:

  • Gathers detailed system hardware and software information.
  • Creates a comprehensive system profile for auditing.

Why it's important:

  • Provides a baseline for system audits.
  • Assists in troubleshooting hardware and software issues.
  • Documents system configuration for future reference.

When to use:

  • Monthly.
  • After making hardware changes.
  • Before major system updates.

Best Practices

Regular Monitoring

  • Daily: Check the Dashboard and System Status for any anomalies.
  • Weekly: Review recent logs and security events.
  • Monthly: Perform comprehensive system audits using all available scans.

Security Scans

  • Run Malware Check daily to catch threats early.
  • Schedule Firewall Analysis and Startup Items Check weekly.
  • Perform all Core Security Scans monthly for thorough security assessments.

Log Management

  • Regularly review logs in the Log Analysis section.
  • Archive important logs for long-term storage.
  • Delete outdated or unnecessary logs to conserve disk space.

LLM Analysis

  • Use specific, clear questions when requesting AI insights.
  • Combine multiple logs for comprehensive analysis.
  • Validate AI recommendations with your own knowledge or consult a professional.

Troubleshooting

Common Issues

Scan Failures

  • Ensure you have the necessary permissions (run scans with admin rights).
  • Verify that you have sufficient disk space and system resources.
  • Check logs for error messages and consult the FAQ or support.

LLM Issues

  • Confirm the AI model is correctly installed and configured.
  • Ensure your system has enough RAM (16GB minimum recommended).
  • Restart the application if the AI module becomes unresponsive.

Performance Problems

  • Close unnecessary applications to free up system resources.
  • Consider upgrading hardware if performance issues persist.
  • Regularly clear old logs and temporary files.

Connection Errors

  • Ensure the application is running and listening on the correct port (5002 by default).
  • Check your firewall settings to allow connections to the application.
  • Use localhost or 127.0.0.1 in your browser to access the application.

FAQ

General Questions

Q: How often should I run security scans?

A: It's recommended to run basic scans daily and comprehensive scans weekly or monthly, depending on your usage patterns.

Q: Can I customize security checks?

A: Yes, you can modify existing scripts or add new ones in the scripts directory to tailor the scans to your needs.

Q: Is GuardStick compatible with Windows?

A: Currently, GuardStick is designed for Unix-based systems like macOS and Linux.

Technical Questions

Q: What permissions are required to run GuardStick?

A: Administrator privileges are needed for system-level operations and to perform comprehensive security scans.

Q: Can I use a different LLM model?

A: Yes, you can configure a different model by modifying the MODEL_PATH in the .env file and updating the llm_api.py script accordingly.

Q: How are logs stored and secured?

A: Logs are stored in the data/log_reports directory. Sensitive data is encrypted to protect your privacy.
